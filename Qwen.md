# Qwen.md - Инструкции для разработки игры

## Обзор проекта
Мультиплатформенная РПГ на Java с использованием libGDX для Android и PC.

## Ключевые механики

### 1. Глобальная гексагональная карта
- **Тип карты**: Гексагональная сетка (hex grid)
- **Перемещение**: 
  - Пошаговое перемещение в стиле roguelike
  - Учёт скорости персонажей (разное количество гексов за ход)
  - Подсветка доступных для перемещения гексов
  - Расчёт пути с учётом препятствий и стоимости перемещения по разным типам местности
- **Координаты**: Использование кубических или осевых координат для гексов
- **Отрисовка**: Изометрическая или плоская проекция гексов

### 2. Переход на локальную карту
- **Триггер перехода**: 
  - Вход в определённую ячейку (город, подземелье, здание)
  - Специальное действие игрока при нахождении в нужной ячейке
- **Загрузка локации**: 
  - Динамическая загрузка данных локации
  - Сохранение состояния глобальной карты
  - Возможность возврата на глобальную карту из специальных точек выхода
- **Масштабирование**: Плавный переход между масштабами карт

### 3. Окно крафта
- **Интерфейс**: 
  - Список доступных рецептов
  - Отображение требуемых ресурсов
  - Прогресс создания предмета
  - Фильтрация и поиск рецептов
- **Логика**:
  - Проверка наличия ресурсов в инвентаре
  - Учёт навыков персонажа для сложных рецептов
  - Возможность массового крафта
  - Система улучшения предметов через крафт

### 4. Окно персонажа
- **Отображаемая информация**:
  - Характеристики (сила, ловкость, интеллект и т.д.)
  - Уровень и опыт
  - Здоровье, мана, выносливость
  - Экипировка (визуальное отображение слотов)
  - Навыки и умения
  - Инвентарь с возможностью сортировки
- **Функционал**:
  - Распределение очков характеристик
  - Изучение новых навыков
  - Смена экипировки
  - Просмотр эффектов статуса

### 5. Пошаговые бои на локальной карте
- **Режим боя**:
  - Отдельная боевая сцена с тактической картой
  - Пошаговая система (игрок -> враги -> игрок)
  - Инициатива на основе характеристик персонажей
- **Боевая механика**:
  - Различные типы действий (атака, защита, умение, предмет, перемещение)
  - Система укрытий и высоты местности
  - Зоны поражения для массовых атак
  - Статусные эффекты (оглушение, яд, горение и т.д.)
- **ИИ противников**:
  - Разумное поведение врагов
  - Приоритеты целей
  - Использование окружения
- **Завершение боя**:
  - Победа/поражение/побег
  - Расчёт награды и опыта
  - Возврат на глобальную или локальную карту

## Технические требования

### Архитектура
- **Core module**: Общая логика игры (Java)
- **Desktop launcher**: Запуск для PC
- **Android launcher**: Запуск для Android
- **Asset management**: Централизованная загрузка ресурсов
- **Save system**: Сохранение прогресса в кроссплатформенном формате

### Управление состоянием
- **Game States**: Меню, глобальная карта, локация, бой, крафт, персонаж
- **Event system**: События для взаимодействия между системами
- **Data persistence**: JSON/XML для сохранения данных игры

### UI/UX
- **Адаптивный интерфейс**: Поддержка разных разрешений экранов
- **Управление**: 
  - Мышь/клавиатура для PC
  - Тач-интерфейс для Android
- **Визуальные эффекты**: Анимации переходов, частицы, подсветки

## Структура данных

### Гексагональная карта
```
Hex {
  q, r, s: координаты
  terrain: тип местности
  elevation: высота
  objects: список объектов
  isExplored: исследована ли
}
```

### Персонаж
```
Character {
  stats: характеристики
  inventory: инвентарь
  equipment: экипировка
  skills: навыки
  currentHex: текущая позиция
  movementPoints: очки перемещения
}
```

### Боевая система
```
Battle {
  turnOrder: порядок ходов
  combatants: участники боя
  map: боевая карта
  currentTurn: текущий ход
  actions: доступные действия
}
```

## План реализации

### Этап 1: Базовая инфраструктура
- Настройка проекта libGDX
- Создание основных экранов и переходов
- Система сохранения/загрузки

### Этап 2: Глобальная карта
- Реализация гексагональной сетки
- Система перемещения с учётом скорости
- Генерация/загрузка карты мира

### Этап 3: Локации и переходы
- Система локальных карт
- Механика перехода между масштабами
- Базовые взаимодействия в локациях

### Этап 4: Интерфейсы
- Окно персонажа
- Окно крафта
- Инвентарь и управление предметами

### Этап 5: Боевая система
- Тактическая боевая карта
- Пошаговая система боя
- ИИ противников
- Навыки и способности в бою

### Этап 6: Полировка
- Баланс игровой механики
- Оптимизация производительности
- Тестирование на обеих платформах

## Примечания для разработки
- Всегда проверять совместимость с Android и Desktop
- Использовать ассет-менеджер для эффективной загрузки ресурсов
- Реализовать систему логирования для отладки
- Создать инструменты для редактирования карт и баланса
- Документировать все изменения в коде
